# Cochrane Master Builders
## Universal Website Template Wireframe Plan

---

## Purpose Of This Wireframe System

This document is the base website template plan for the full Cochrane Master Builders service
ecosystem.

The goal is to create one premium, consistent, SEO-ready, performance-optimized website
structure that can be reused across all service-specific domains while still feeling custom to
each service category.

Every website should feel like it belongs to the same family-owned builders’ network, but each
one should still feel tailored to the service: roofing should feel protective and durable, tile should
feel precise and crafted, basement suites should feel safe and income-focused, landscaping
should feel outdoor and lifestyle-driven, and custom homes should feel aspirational and
legacy-focused.

The template must support:

- The same master brand identity across all domains.
- A consistent navigation system.
- A consistent contact form.
- A strong home hero section.
- A strong service-specific homepage template.
- A reusable service page structure.
- A reusable pricing section.
- A reusable areas-we-serve section.
- A reusable SEO-focused area landing page system.
- Fast loading, mobile-first performance.
- Simple content swapping for all 115 service websites.

---

## Core Template Philosophy

The system should not feel like 115 random websites.

It should feel like one high-trust construction network with 115 highly focused entry points.

The customer may land on CochraneRoofingMasters.ca, CochraneBathroomRenoMasters.ca,
CochraneTileMasters.ca, or any other domain. The layout should immediately tell them:
1. This is specific to exactly what I searched for.
2. This company understands this exact service.
3. This is connected to a serious local builder.
4. This feels safe, clear, and professional.
5. I can understand the process and rough pricing before I call.
6. I can trust them because the brand feels family-owned, local, and long-term.

The template should be clean and simple, but deep.

The visual language should not be cluttered. The depth should come from clear sections, useful
content, pricing transparency, structured FAQs, proof, service details, and strong calls to action.

Every site should carry the same emotional foundation:

**Strong Foundations For Those Who Come After Us.**

This should appear in some form on every website, either in the hero subtext, about section,

## Performance Optimization Requirements

Performance must be baked into the wireframe.

### Targets

- LCP under 2.5 seconds
- CLS under 0.1
- INP under 200ms
- Mobile-first experience
- No heavy homepage sliders
- No autoplay video above the fold
- Minimal third-party scripts

### Image Rules

- Hero image must be compressed and sized properly.
- Use AVIF or WebP.
- Lazy-load all below-fold images.
- Use responsive image sizes.
- Never load full gallery images on first page load.

### Animation Rules

- Use motion sparingly.
- Scroll animations should support clarity, not overwhelm.
- Prefer opacity, transform, and subtle reveal effects.
- Avoid layout-shifting animations.
- Respect reduced-motion settings.

### Form Rules

- Contact form should load fast.
- Use lightweight validation.
- Do not depend on heavy third-party form embeds if avoidable.
- Store domain/source info with hidden fields.

### Code Architecture

Recommended stack:

- React 18
- Vite
- Static-first pages where possible
- Shared component library
- JSON/content config per domain
- Lightweight routing
- Minimal JS for above-fold rendering
