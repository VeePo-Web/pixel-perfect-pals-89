## 2. What this document is

The brand-identity standard for CMB's family-legacy positioning. Governs the *language* of family, generations, craft passed down, and what it means to "build for those who come after us." Source of truth for tone words, voice guardrails, and the visual register that supports the legacy positioning (palette philosophy, type philosophy).

## 8. Rules — non-negotiables this doc enforces

1. **Heritage register, never startup register.** No "ship," "launch," "stack," "platform," "powered by," "AI-driven," "next-gen." (source: voice section + anti-pattern list)
2. **Family language is earned, not sprinkled.** "Generations," "legacy," "passed down" appear when the surrounding copy substantiates them — never as decoration. (source: family-language register)
3. **Tone arrays in `trade.config.ts` derive from this doc.** When updating a trade's voice arrays, cite the section consulted; never freelance vocabulary.
4. **Palette derivation, not palette invention.** A remix accent derives from the master palette via the rules in this source (hue shift, saturation band, contrast preservation). Never invent a palette from a stock generator. (source: visual register)
5. **Type pairs respect the heritage register.** Display + body must read as bespoke editorial, never as default-system or trendy variable-font flex. (source: type register)
6. **Capitalization discipline.** Title Case for headlines; sentence case for sub-heads and body. ALL-CAPS reserved for eyebrows ≤ 4 words. (source: voice section)
7. **No emoji in product copy.** Ever. (source: anti-pattern list)
8. **No exclamation marks in headlines or body.** Quiet confidence. (source: voice section)
9. **Numbers spelled or numeralized per register.** Years and counts (decades, generations, "since 2014") use numerals; abstract counts ("a dozen reviews") spell. (source: voice section)
10. **Voice adapts per persona but never inverts.** Subcontractor B2B can be more direct; mother persona can be more reassuring; grandfather persona can be more plainspoken — but no persona overrides the heritage register's word ban. (cross-link to personas)
11. **Palette accents pass WCAG contrast on real surfaces.** A derived accent that fails contrast on the master neutral surfaces is rejected. (cross-link to `gr-wcag-aa`)
12. **The on-brand sanity check is mandatory.** Before any hero copy or palette ships, run the AI prompt set in §11 of this doc and confirm zero violations.

## 9. Anti-patterns

- A hero headline using "ship" or "launch" — violates Rule 1.
- "We're a fourth-generation builder" copy on a brand-new remix where there is no fourth generation. Violates Rule 2.
- Trade-config voice array invented to sound "fresher" than the master. Violates Rule 3.
- A drywall remix accent generated from coolors.co with no derivation chain to the master. Violates Rule 4.
- System UI sans-serif body font chosen "for performance." Violates Rule 5.
- Hero "BUILT TO LAST." Violates Rules 6 + 8.
- "Call us today! 🏠✨" — violates Rules 7 + 8.

## 10. Worked example

**Trigger.** Request: "Pick voice arrays + palette accent for the new Roofing remix's `trade.config.ts`."

**Step 1 — recognize.** "Voice arrays" + "palette accent" + "remix" → this doc primary; `strategy/1.3.partner.md` for inheritance mechanics.

**Step 2 — read source.** Parse PDF. Read voice section + visual register + family-language register.

**Step 3 — apply rules.** Rule 1: ban startup vocabulary in the voice array. Rule 3: cite the voice section in the plan. Rule 4: derive accent via the source's rules (e.g. shift master accent hue by the prescribed band toward a copper that signals roofing without breaking contrast). Rule 11: verify accent passes WCAG against master neutrals. Rule 5: pick a type pair consistent with master register; no new pair invented.

**Step 4 — decision.** Voice arrays: heritage adjectives, plainspoken verbs, no startup nouns. Accent: derived hex with contrast verified. `trade.config.ts` updated, plan cites this doc + 1.3 (inheritance) + checks `gr-bespoke-brand-derivation`, `gr-bespoke-style-guide-live`, `gr-wcag-aa`.

**Step 5 — verify.** Run the §11 prompt set against the source, confirm zero violations.

## 11. AI prompts to run against the source

1. "Extract every banned word / phrase listed in the source, grouped by category (startup vocabulary, exaggerations, trendy verbs, etc.)."
2. "Extract every sanctioned tone word or adjective, grouped by usage context (hero, sub-hero, body, eyebrow)."
3. "Extract the palette derivation rules (hue, saturation, lightness bands; contrast requirements) as a procedure."
4. "Extract the type pair philosophy: display register, body register, fallback rules, weight discipline."
5. "Extract the family-language register: which words are sanctioned, which require substantiation, which are forbidden."
6. "Extract any anti-patterns the source lists explicitly."
