## 2. What this document is

The UX design-phase contract for CMB and its sister sites. Defines the bespoke / traditional / editorial register that every CMB-family site must hit: layout density, hierarchy choices, when to choose restraint over flash, mobile UX defaults, trust-signal placement, footer architecture, form structure, and the patterns that distinguish "bespoke craft" from "templated polish."

## 8. Rules — non-negotiables this doc enforces

1. **Bespoke register, never template register.** No rounded card grids of 3 centered icons with one-line labels; no "stat counters" that animate up; no testimonial carousels with author headshots. (source: anti-pattern section)
2. **One primary action per viewport.** Hero gets one CTA; secondary action is visually subordinate. (source: hierarchy section)
3. **Density follows the editorial scale.** Body 16–18px / line-height 1.6+; display sizes per the source's scale; never use system defaults. (source: typography)
4. **Touch targets ≥ 44×44 CSS px on mobile.** ≥ 48×48 when grandfather persona is primary. (source: responsive section + cross-link to `personas-icp/1.4.3`)
5. **Safe-area insets honored on iOS.** Sticky CTAs, modals, and footers respect `env(safe-area-inset-*)`. (source: responsive section)
6. **Footer is a tiered architecture.** Top brand + CTA tier, middle nav columns, bottom monumental brand + legal. Never a single flat row of links. (source: footer architecture)
7. **Real NAP in the footer, on every page.** Name, address, phone — visible (not modal-gated). (source: trust section + cross-link to `gr-real-business-signals`)
8. **Trust signals appear within 1 scroll on every conversion page.** License, insurance, warranty, real reviews. (source: trust section + cross-link to `gr-real-business-signals`)
9. **Forms ≤ 4 visible fields per step.** Multi-step preferred over a wall of fields. (source: form section)
10. **Forms submit within 60 seconds on mobile for the primary booking path.** (source: form section + cross-link to `gr-booking-one-tap`)
11. **Reviews are real, attributed, and dated.** No testimonial carousels of stock quotes. (source: trust section)
12. **Sticky CTA is allowed on mobile, never auto-popups.** No modal interrupts the first viewport. (source: anti-pattern section)
13. **Hero image direction: subject + restraint.** No people; macro detail of craft preferred; gradient overlays preserve text contrast. (source: visual direction)
14. **No layout shift on load.** Reserve aspect-ratio for images; preload heroes; `fetchpriority="high"` on LCP. (cross-link to `gr-performance-budget-mobile`)

## 9. Anti-patterns

- Three centered cards each with a circle icon + 4-word title + one-sentence description. Violates Rule 1.
- Hero with three equal-weight buttons. Violates Rule 2.
- Body text at 14px / line-height 1.4. Violates Rule 3.
- 38×38 touch target for the booking CTA on mobile. Violates Rule 4.
- Sticky CTA hiding behind the iOS home indicator. Violates Rule 5.
- Footer = single row of 6 links + copyright. Violates Rule 6.
- Phone number only revealed on click. Violates Rule 7.
- License + insurance only on the About page footnote. Violates Rule 8.
- Booking form with 11 visible fields on step 1. Violates Rule 9.
- Auto-modal "Get a free quote!" appearing 3 seconds after load. Violates Rule 12.
- Testimonial carousel of 6 stock quotes attributed to "Sarah M." with no date. Violates Rule 11.
- Hero image of a smiling family in front of a house. Violates Rule 13 (people).

## 10. Worked example

**Trigger.** Request: "Build the booking-form step UI for the home page modal."

**Step 1 — recognize.** Form structure + mobile defaults + trust placement → this doc primary; `personas-icp/1.4.2.partner.md` for mother-persona scheduling sensitivity.

**Step 2 — read source.** Parse DOCX. Read form section + responsive section + trust section.

**Step 3 — apply rules.** Rule 9: ≤ 4 fields per step (split into 4 steps). Rule 10: 60s mobile target. Rule 4: 48×48 touch targets. Rule 5: safe-area inset on the sticky submit. Rule 8: license + insurance trust strip visible inside the modal. Rule 12: no auto-popup.

**Step 4 — decision.** 4-step modal: contact / job type / scope / confirm. Each step ≤ 4 fields. Trust strip in the bone-panel region (per existing memory of two-region modal). Sticky bottom-bar submit with safe-area padding. Submit-time animation per existing brand memory.

**Step 5 — verify.** Plan §12 lists `gr-booking-one-tap`, `gr-real-business-signals`, `gr-wcag-aa`, `gr-performance-budget-mobile`.
