# Purpose

The single canonical reference for the entire Cochrane Master Builders ecosystem — the umbrella hub plus every planned service-specific spin-off site. It defines brand naming, domain pattern, cluster structure, hub↔spoke SEO map, launch phasing, and per-site SEO targets. Treat this as the **remix base**: every new service-specific Cochrane site is generated by pulling its row out of the Master Tracker and dressing it in the brand identity, design persona, and motion layers stored elsewhere in `/knowledge/`.

# What it contains (6 sheets)

1. **00 Audit Summary** — purpose, limitations (notably: live `.ca` availability NOT verified), main inbox, brand promise.
2. **01 Master Tracker** — 116 rows × 24 cols. The catalog of every planned site: brand/site name, recommended `.ca` domain, canonical hub URL slug, primary SEO keyword, secondary keywords, buyer intent, offer type, revenue potential, lead urgency, launch priority (P0 Hub → Pn), cluster, status, and family-legacy column.
3. **02 Service Clusters** — 21 clusters with priority role + first-wave services + SEO notes.
4. **03 Hub SEO Map** — internal-link/anchor-text architecture between the hub and every specialist site.
5. **04 Domain Rules** — naming pattern (`Cochrane + Service + Masters + .ca`), spelling rules, anti-patterns, rationale.
6. **05 Launch Roadmap** — Phase 0 foundation through later phases, with goals and explicit launch criteria per phase.

# What this document influences

- **Domain selection** — must follow Sheet 04 pattern. Reject any non-conforming domain (no random brand names, no missing "Cochrane" or "Masters", no `.com` substitutions unless explicitly approved).
- **Brand / site naming** — every spin-off name comes from Sheet 01.
- **SEO targeting** — primary keyword + secondary keyword set + buyer-intent + urgency framing for each site come from Sheet 01.
- **Information architecture** — cluster grouping (Sheet 02) drives content cross-linking, related-services modules, and IA on the hub directory.
- **Internal linking + anchor text** — every hub→spoke and spoke→hub link uses the anchor + purpose defined in Sheet 03 ("Cochrane [Service] Masters" outbound, "Part of Cochrane Master Builders" inbound).
- **Launch order** — Sheet 05 phasing is binding. Phase 0 foundation work (hub live, inquiry form working, Search Console verified) must be complete before any specialist site goes live.
- **Trust signals** — every specialist site must carry the "Part of Cochrane Master Builders" attribution in header/footer per Sheet 03.

# Adaptation rules (firewall)

- **Brand context is Cochrane Master Builders, NOT Masters Concierge / VeePo.** Do not apply the Masters/VeePo automotive-detailing vocabulary firewall to anything sourced from this doc. It operates exclusively under `brands/cochrane-master-builders/brand-identity/...`.
- **`.xlsm` is ground truth.** If a future Cochrane brand-identity, strategy, or persona doc conflicts with the spreadsheet, surface a memory-conflict flag — do not silently override.
- **Live `.ca` availability was NOT verified.** Whenever the system recommends a domain pulled from the Master Tracker, repeat this caveat and recommend a WHOIS check before purchase.
- **Structural plan, not visual direction.** This file does not dictate colors, typography, motion, or component styling — those still route through the Cochrane brand identity, design persona, and motion personas. Do not invent visual direction from this doc.
- **Family-legacy framing.** The "family legacy" column is a positioning lens (multi-generational craftsmanship, "foundations for generations after us"). Use it to inform tone when writing copy for any spin-off, not as literal copy.
