## Main Purpose
Canonical per-site **UX theme style guide** for all 115 planned Cochrane spin-off sites. The "brand-feel + surface treatment + component tone" leg of the remix **octapod** (eight paired files):

- **v1.0** — SEO / positioning / linking / launch phase (`.xlsm`)
- **v2.0** — Services + mid-high CAD pricing + compliance guardrails
- **v4.0** — Per-site Ideal Customer brain (voice, fears, drivers, intent)
- **v5.0** — Per-site color + UX direction (palette, mood, rationale)
- **v6.0** — Per-site UX experience (pacing, motion temperament, emotional arc)
- **v7.0** — Per-site performance audit specs (CWV, asset/render/mobile)
- **v9.0** — Per-site theme layout + bespoke UX components + perf-aware build notes
- **v10.0 (this file)** — Per-site **UX theme style guide** (brand feel, mood, surface treatment, typographic posture, motif language, component tone)

All eight join on `Master Row`. Drift in `Recommended Domain` between any two = **Sync Discrepancy**.

## Sheet Inventory
1 sheet · 116 rows × 5 cols.

`UX Theme Style Guides`:
1. **Master Row** — join key.
2. **Recommended Domain** — must stay in sync across all paired files.
3. **Website / Brand** — display name.
4. **Cluster** — service cluster (e.g., Umbrella / Authority).
5. **UX Theme Style Guide** — per-site narrative covering brand feel, mood, surface treatment, motif language, and component tone.

## What This Document Should Influence
- Per-site **brand-feel + look-and-feel** narrative (calm/deep/clean/premium-without-flashy, etc.).
- **Surface treatment** — paper vs. matte, grain, tonal warmth, light behavior.
- **Typographic posture** — display weight, body restraint, cap-height intentions.
- **Motif language** — recurring shapes, line weights, frame motifs, photography crops.
- **Component tone** — button silhouette, card edges, divider behavior, hover quietness.
- Cluster-level family resemblance — Umbrella / Authority sites share style cues; service-specific spin-offs vary tone within the family.

## Adaptation Rules / Firewall
- Brand context = **Cochrane Master Builders** only. No automotive vocabulary firewall.
- These are **directional style narratives**, not finished tokens or specs. Reconcile against Cochrane brand-identity (`1.2.1`, `1.2.2`), Cochrane UX (`1.3.1`), v5.0 color, v6.0 experience temperament, v9.0 layout, and v7.0 perf budget before shipping any actual style decision.
- **Per-site only** — never blend rows. Always look up by `Master Row`.
- **Cluster-aware:** rows in the same `Cluster` may share style DNA; differentiate via v4.0 ICP voice + v5.0/v6.0 mood and temperament.
- **Join discipline:** any per-site work must reference the same `Master Row` across all paired files.
- **Backend intelligence only.** Embedding does not change any front-end output. VeePo / Masters Detailing front-end stays untouched and no Cochrane spin-off site exists in this codebase yet.

## Mode-OS Pairing
- **Architect** — uses the row's style guide when scaffolding the spin-off's design tokens, surface treatments, and component silhouettes.
- **Mapper** — renders per-cluster style matrices (mood × motif × surface).
- **Auditor** — verifies built sites against (a) the row's stated brand feel / motif language and (b) v5.0 palette + v6.0 temperament + v9.0 layout + locked brand-identity memories. Style drift from stated mood = **High** bug; violation of brand-identity 1.2.1/1.2.2 = **Critical**.
