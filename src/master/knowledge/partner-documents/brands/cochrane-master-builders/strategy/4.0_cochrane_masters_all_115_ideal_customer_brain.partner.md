## Main purpose

Canonical **per-site Ideal Customer Profile (ICP) brain** for the Cochrane Master
Builders family of sites. Supplies the *voice + emotion + intent* leg of the
remix tripod. The set joins on **`Master Row`**:

- **v1.0 Master Plan** — positioning, SEO, hub↔spoke linking, launch phase.
- **v2.0 Services & Pricing** — offers, mid-high CAD ranges, compliance guardrails.
- **v4.0 ICP Brain (this file)** — *who you are talking to and how to speak to them.*

All three files **must be consulted together** for any new spin-off site. Drift
in `Recommended Domain` between files is a Sync Discrepancy and should be flagged
by the Auditor before shipping.

---

## What this document influences

- Hero copy + sub-hero promise on every spin-off site.
- Section headlines, subheads, microcopy, value-prop framing.
- Per-site tone of voice (layered on top of the cross-brand archetype docs —
  `1.4.1` Subcontractor / `1.4.2` Mothers / `1.4.3` Grandfathers).
- Trust language, fear-reversal copy, objection handling.
- Form labels, CTA verbs, success / confirmation states.
- FAQ topic selection and phrasing (pairs with the FAQ-SEO persona).
- Image direction prompts — mood, subject, scene choice (still bound by the
  brand's image rules).
- Lead-qualification *feel* of the quote form, not just its fields.

## Adaptation rules / firewall

- Brand context = **Cochrane Master Builders** only. **No** automotive
  vocabulary firewall (this is the construction brand, not VeePo / Masters).
- The narratives are **per-site**, not a master ICP — never blend rows.
  Always look up the row by `Master Row` and stay inside it.
- Use the row's narrative as the **voice source**, not as ship-ready copy.
  All output copy must still pass the cross-brand storytelling / round-two
  copywrite persona and the brand's voice memories before going live.
- Cross-brand Cochrane ICP archetype docs (`1.4.1` Subcontractor,
  `1.4.2` Mothers, `1.4.3` Grandfathers) sit **above** this file when their
  archetype is the primary buyer for the site — this file is the per-site
  nuance layer on top, not a replacement.
- **Join discipline:** any per-site copywriting work must reference the same
  `Master Row` across v1.0 + v2.0 + v4.0. If `Recommended Domain` differs
  between the three files for the same row, raise a Sync Discrepancy.
- Snapshot status: treat the narratives as *"indicative ICP, audited at embed
  time."* If a more recent ICP doc contradicts a row, surface the conflict
  rather than silently overriding.
- This document is **backend intelligence only**. Embedding it must not change
  any front-end output on its own.

## Mode-OS pairing

- **Architect** — uses the row's ICP narrative when designing the IA / page
  structure for a new spin-off site (drives section ordering, what to put
  above the fold, what objections to address before the CTA).
- **Mapper** — can render per-cluster ICP affinity diagrams or per-site
  emotional-journey flows (awareness → trust → quote request).
- **Auditor** — pressure-tests generated copy against (a) the row's ICP
  narrative (does the headline actually speak to *this* person?) and
  (b) the join across v1.0 / v2.0 / v4.0. Tone misalignment with the
  row's ICP narrative = **High** bug.

## Practical examples

- *"Write the hero for the Basement Suite Masters site."* → Pull that row's
  ICP narrative as voice + emotional-driver source; structure the headline
  around the stated WHO / WANTS / FEARS, then route through the storytelling
  persona for final wording.
- *"What FAQs should this site have?"* → Mine the row's narrative for fears
  and objections, then route through the FAQ-SEO persona for structure +
  schema.
- *"Pick a hero image direction for this site."* → Use the ICP narrative to
  set mood and subject; image rules from the Cochrane brand-identity docs
  still bind.
- *"Tone-check this draft against the buyer."* → Compare draft against the
  row's ICP narrative; surface mismatches as Auditor findings before ship.
