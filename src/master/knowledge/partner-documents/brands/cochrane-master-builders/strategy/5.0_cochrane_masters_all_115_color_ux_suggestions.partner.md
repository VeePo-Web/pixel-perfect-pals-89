## Main purpose

Canonical **per-site palette + visual mood + UX rationale** for the Cochrane
Master Builders family of sites. The "color + feel" leg of the remix tetrapod:

- **v1.0 Master Plan** — positioning, SEO, hub↔spoke linking, launch phase.
- **v2.0 Services & Pricing** — offers, mid-high CAD ranges, compliance.
- **v4.0 ICP Brain** — per-site voice, emotion, intent.
- **v5.0 Color + UX Direction (this file)** — *what each site should look and feel like.*

All four files **must be consulted together** for any new spin-off site.
Drift in `Recommended Domain` between any of the four for the same `Master Row`
is a Sync Discrepancy and should be flagged by the Auditor.

---

## What this document influences

- Per-site color tokens (primary / secondary / surface / accent / text) on
  every spin-off site.
- Hero mood, section backgrounds, divider treatments.
- Image art-direction: tonal grading, scene mood, material focus.
- Button + CTA accent color choice (still bound by Cochrane-wide CTA rules).
- Background imagery selection — heritage forest vs. concrete-gray vs. warm
  bone vs. quarry-stone, etc.
- Section-divider gradients, texture choices, surface depth.

## Adaptation rules / firewall

- Brand context = **Cochrane Master Builders** only. **No** automotive
  vocabulary firewall (this is the construction brand, not VeePo / Masters).
- These are **directional palette + mood suggestions**, not finalized tokens.
  They must be reconciled against the cross-brand Cochrane brand-identity
  docs (`1.2.1` Family Legacy Standard, `1.2.2` Foundations) before being
  shipped as `tailwind.config.ts` / CSS variables.
- Use the row's narrative as the **mood + grouping source**, not as ship-ready
  hex codes. Final tokens still pass through the Brand Identity Architect
  persona and the master design persona for QA.
- **Per-site only** — never blend rows. Always look up by `Master Row` and
  stay inside it.
- **Join discipline:** any per-site visual work must reference the same
  `Master Row` across v1.0 + v2.0 + v4.0 + v5.0. If `Recommended Domain`
  differs between the four files for the same row, raise a Sync Discrepancy.
- This document is **backend intelligence only**. Embedding it must not
  change any front-end output on its own — VeePo / Masters Detailing front-end
  is untouched and no Cochrane spin-off sites currently exist in this
  codebase.

## Mode-OS pairing

- **Architect** — uses the row's palette + mood when scaffolding tokens /
  theme variables for a new spin-off (drives `--background`, `--surface`,
  `--accent`, etc., and the hero mood spec).
- **Mapper** — can render per-cluster palette affinity diagrams or
  palette-vs-archetype matrices to find visual coherence across the family.
- **Auditor** — pressure-tests generated visuals against (a) the row's
  color rationale (does the mood actually match?) and (b) the join across
  all four files (does the palette serve this ICP and this service mix?).
  Mood/palette mismatch with the row's UX rationale = **Medium**–**High** bug.

## Practical examples

- *"Generate the Tailwind palette for the Basement Suite Masters site."*
  → Pull that row's color grouping + UX rationale, reconcile against Cochrane
  brand-identity tokens, propose final HSL tokens via the Brand Identity
  Architect persona.
- *"Pick a hero background mood for this site."* → Use the row's theme + mood;
  pair with v4.0 ICP narrative for emotional alignment; respect Cochrane image
  rules.
- *"Audit this spin-off's color choices."* → Compare current tokens vs. the
  row's UX rationale; surface mismatches as Auditor findings before ship.
- *"Suggest a section divider treatment for this site."* → Use the row's
  texture / material feel guidance; respect any global Cochrane divider rules.
