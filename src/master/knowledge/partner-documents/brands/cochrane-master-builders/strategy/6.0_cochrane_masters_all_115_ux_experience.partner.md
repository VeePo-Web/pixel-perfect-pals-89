## What this document is

The fifth paired companion in the Cochrane family-of-sites system. Joined on `Master Row` to:

- **v1.0** Master Plan — positioning + SEO + linking + launch phase.
- **v2.0** Services & Pricing — services offered + mid-high CAD ranges + compliance guardrails.
- **v4.0** Ideal Customer Brain — voice + emotional drivers + fears + language.
- **v5.0** Color + UX Direction — palette + visual mood + UX rationale.
- **v6.0 (this file)** UX Experience — per-site **UX Experience Theme + Experience Strategy**: how the site should *feel and behave* under interaction.

1 sheet (`UX Experience`), 116 rows × 3 columns: `Master Row`, `Recommended Domain`, `UX Experience For Each One`.

## Purpose

Canonical per-site **interaction temperament + experience strategy**. The "how it behaves and feels under the cursor" leg of the remix pentapod. Distinct from v5.0:

- **v5.0 = what it looks like** (palette, mood, visual direction).
- **v6.0 = what it feels like to use** (pacing, rhythm, motion temperament, trust-building cadence, emotional arc).

## What this document influences

- Page rhythm and pacing (reveal speed, IA density vs. breathiness).
- Motion temperament (calm-control · confident-craft · quiet-authority · etc.).
- Section ordering and emotional arc (where trust, proof, and friction-reducers land).
- Hover / scroll / transition character — easing curves, durations, restraint level.
- Density per fold; negative-space lean vs. proof-density lean.
- Form behavior tone (gentle nudges vs. crisp directness).
- Loading / empty / success / error state personality.

## Triggers — when AI should consult this

Any prompt touching: *UX, experience, feel, vibe, pacing, rhythm, motion temperament, interaction style, scroll behavior, hover personality, transitions, micro-interactions, density, breathing room, section ordering, emotional arc, trust pacing, "how should this site feel," loading/empty/success state personality, form temperament.*

Always look up the relevant `Master Row` first; never blend rows.

## Adaptation rules / firewall

- **Brand context = Cochrane Master Builders only.** No automotive vocabulary firewall (Cochrane is construction, not VeePo / Masters Detailing).
- **Directional, not finished specs.** These are UX *themes*, not motion specs. Reconcile against:
  - Cochrane brand-identity: `1.2.1` Family Legacy Standard, `1.2.2` Foundations.
  - Cochrane UX: `1.3.1` Bespoke / Traditional UX Design Phase Report.
  - Cross-brand motion / scroll-craftsperson personas.
- **v5.0 vs. v6.0 split — both required.** v5.0 governs palette + visual mood; v6.0 governs interaction temperament + experience pacing. Never let one silently override the other on a new spin-off.
- **Per-site only.** Always look up by `Master Row`. Never blend per-site narratives.
- **Join discipline.** Any per-site UX work must reference the same `Master Row` across **v1.0 + v2.0 + v4.0 + v5.0 + v6.0**. Drift in `Recommended Domain` between any of the five for the same `Master Row` = **Sync Discrepancy** (surface to user, do not silently reconcile).
- **Backend intelligence only.** Embedding this document does not change any front-end output on its own. VeePo / Masters Detailing front-end stays untouched. No Cochrane spin-off sites currently exist in this codebase.

## Mode-OS pairing

- **Architect** — uses the row's experience theme + strategy when scaffolding IA, section order, and motion spec for a new spin-off.
- **Mapper** — can render per-cluster experience-temperament matrices (calm-control vs. confident-craft vs. quiet-authority) or per-site emotional-journey flows.
- **Auditor** — pressure-tests generated UX against (a) the row's experience theme — *does the page actually feel like this?* — and (b) the join across all five files. Temperament mismatch with the row's UX strategy = **Medium**–**High** bug.
