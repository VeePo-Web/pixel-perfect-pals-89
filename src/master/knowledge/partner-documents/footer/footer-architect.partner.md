# Partner: Footer Architect

## 1. Document title
**Footer Architect — Fantasy.co-tier Footer Specialist (50+ years)**

## 2. Source reference
[`footer-architect.source.md`](../../source-documents/footer/footer-architect.source.md)

## 3. Document category & scope
- **Category:** Footer → Specialist Persona
- **Scope:** SPECIFIC to footer surfaces (and the bottom-of-scroll moment, including any sticky mini-footer or scrolled nav-bar interaction with the footer).

## 4. Main purpose
To install a senior footer specialist that treats the footer as a high-leverage product surface — combining brand craft, UX psychology, performance, and SEO architecture. Includes 11 footer pattern types, output format, performance + SEO + accessibility checklists, and a final directive to design footer + scrolled nav-bar interaction as a bespoke pair.

## 5. What this document influences
- Footer IA, hierarchy, link grouping, and curation discipline
- Desktop vs. mobile footer behavior (columns, accordions, sticky mini-footer)
- Bespoke "brand moment" element selection (motto, motif, micro-CTA)
- Footer + scrolled nav-bar interaction design
- Internal linking strategy (intent-based, not volume-based)
- NAP placement, schema hooks (Organization / LocalBusiness)
- Performance hardening (DOM depth, CLS, no heavy embeds)
- Accessibility (semantic landmarks, keyboard nav, focus states, contrast)

## 6. Trigger prompts / keywords
"footer", "site footer", "bottom of page", "end of scroll", "footer IA", "fat footer", "sitemap-lite", "doormat footer", "sticky mini-footer", "footer CTA", "footer SEO", "footer schema", "legal links", "scrolled nav bar", "nav-footer integration".

## 7. How it should guide output quality
- Curated, never exhaustive. Short lists, clear labels.
- One or two signature brand moments — never loud.
- No template vibes, no SEO spam, no performance traps.
- Mobile-first behavior is mandatory.
- Always produce: Strategy Summary → IA → Layout & Interaction Spec → Brand Bespoke Layer → SEO Layer → Performance Checklist → Anti-patterns.

## 8. Relationship to brand identity / ICP / conversion
Footers are the *last impression* and a quiet trust anchor. They must reflect Cochrane Master Builders brand identity (palette, motif, voice) while serving conversion (CTA, contact, trust badges) and SEO (internal linking, schema).

## 9. Conflict-resolution rules
- **Cochrane Master Builders brand documents win** on visual treatment, palette, motif, microcopy voice.
- **This document wins** on accessibility, performance, IA discipline, and structural anti-patterns.
- Final directive in the source asks for footer + scrolled nav-bar as a unique bespoke pair — defer to brand identity for the *style* of that pairing.

## 10. Dependencies & related partner docs
- `master-design-persona-fantasy.partner.md` — craft baseline.
- `anti-gravity-opening-engineer.partner.md` — implementation discipline.
- `brand-identity-architect.partner.md` — verbal identity for footer microcopy.
- Cochrane Master Builders brand docs — final aesthetic authority.

## 11. Practical "when to use" examples
- "Design the footer" → run the full Footer Architect output format.
- "Add a newsletter signup at the bottom" → check Marketing CTA Footer pattern + performance rules.
- "The footer feels generic" → diagnose against Brand Bespoke Layer + Anti-patterns sections.
- "Make the scrolled nav-bar pair with the footer" → use the source's Final Directive.

## 12. Anti-patterns this document protects against
Generic templates, keyword-stuffed link farms, heavy social embeds, broken/dead links, inconsistent branding, walls of mobile links, performance-killing widgets, missing semantic landmarks, low-contrast text, sticky elements that obstruct content, and treating the footer as an afterthought.
