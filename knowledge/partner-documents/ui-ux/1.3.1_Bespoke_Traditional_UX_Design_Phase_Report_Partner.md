## Main Purpose
This document serves as the **Direct Execution Blueprint for UI and UX**. It translates the overarching "Family Legacy" and "Strategy" into precise, tactile front-end behaviors. It defines the standard for "Bespoke Traditional UX Design"—meaning the website must feel deeply established, rooted, and traditional, while simultaneously operating with the fluid, top-tier interactivity of `fantasy.co` or `igloo.inc`. 

## What This Document Influences
This document acts as an immediate backend decision filter for:
- **Component Styling:** Button UI, spacing, typography hierarchy, border treatments, and layout framing. Every component must look bespoke, never generic or template-based.
- **Interaction & Motion Polish:** Defines how elements react when touched or hovered. Ensures Apple-level motion cleanliness (smooth, intentional, non-jarring).
- **Navigation & Footer Behaviors:** Dictates how the deepest UX frameworks function to hold the user's attention.
- **Sub-Brand Generation (Crucial):** When designing child services (e.g., Cochrane Tile Masters, Drywall Masters), this document sets the master component library rules. The child brand will use this *Bespoke Traditional UX* logic but tailor the imagery/icons to the specific craft.

## When Should the System Consult This?
Code assistants and AI systems must consult this document whenever asked to:
1. Write CSS, Tailwind classes, or styled-components.
2. Build interactive states (hover, focus, active, disabled) for any interactive elements.
3. Design micro-interactions, loaders, or page transitions.
4. Set up the overarching grid, spacing scale, and typography flow.

**Practical Example:** If the user asks "Style the main CTA button" or "Fix the Navigation UI," the AI must consult this document to ensure the button isn't just a flat blue rectangle, but rather a deeply refined, possibly subtly shadowed/textured component with a smooth, premium hover state that honors the family legacy identity.
