ROLE
─────────────────────────────────────────────────────────────────────
You are a performance engineer achieving Lighthouse mobile ≥ 95 /
Accessibility 100 / Best Practices 100 for a React 18 + Vite 5 +
React Router 6 SPA. Performance is a hard business requirement: Google
Maps Pack ranking correlates directly with Core Web Vitals. Every
millisecond you recover is a position in local search. You will make
zero visual changes. Zero copy changes. Only performance plumbing.

─────────────────────────────────────────────────────────────────────
CODEBASE CONTEXT
─────────────────────────────────────────────────────────────────────
Stack: React 18, Vite 5 (vite.config.ts exists), TypeScript, Tailwind 3,
       Framer Motion 12, React Router 6, Supabase JS 2.
Build: vite build → dist/. Served as static files (Lovable/Vercel).
Fonts: Space Grotesk 300/400 + Jost 300/400 via Google Fonts @import in
       src/index.css. Currently blocking, no preconnect.
Images: All template images are AVIF + WebP (generated by image manifest).
        No srcset is wired yet — components use plain <img src="..."/>.
        Areas-we-serve images are external Wikimedia Commons URLs.
Prefetch: PrefetchLink.tsx exists with hover-prefetch for 6 routes.
Scripts: scripts/preflight.ts exists (npm run preflight).
         Add performance gates here, do not create a new script.

Existing slow paths (fix in priority order):
  1. Google Fonts @import — render-blocking, 400–800ms on 4G
  2. No <link rel="preconnect"> for external origins
  3. HeroImage.tsx — <img> without width/height → CLS
  4. EditorialImage.tsx — <img> without aspect-ratio → CLS
  5. BeforeAfterPair.tsx — images without aspect containers → CLS
  6. Framer Motion animating height/top on some components → layout thrash
  7. Lazy boundary gaps on master components

─────────────────────────────────────────────────────────────────────
DELIVERABLES — highest LCP impact first
─────────────────────────────────────────────────────────────────────

STEP 1 — Fix fonts (biggest LCP win, 400–800ms recovery)

  a) Remove the Google Fonts @import from src/index.css.
  b) In index.html, replace with:
       <link rel="preconnect" href="https://fonts.googleapis.com">
       <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
       <link rel="preload" as="style"
             href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@300;400&family=Jost:wght@300;400&display=swap">
       <link rel="stylesheet"
             href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@300;400&family=Jost:wght@300;400&display=swap"
             media="print" onload="this.media='all'">
       <noscript><link rel="stylesheet" href="...same URL..."></noscript>
  c) Add preconnect for all external origins:
       Supabase (from VITE_SUPABASE_URL),
       https://upload.wikimedia.org (areas images),
       https://maps.google.com (map embeds)

STEP 2 — Fix CLS: reserve space for every image surface

  HeroImage.tsx:
    Add width={1920} height={1280}, className="aspect-[3/2]" wrapper,
    fetchpriority="high" decoding="async"

  EditorialImage.tsx:
    Wrap <img> in <div className="aspect-[4/5] overflow-hidden">,
    add width/height props, loading="lazy" decoding="async"

  BeforeAfterPair.tsx:
    Add aspect-[3/2] container per image cell,
    width={1200} height={800}, loading="lazy"

STEP 3 — Create src/components/template/ResponsiveImage.tsx

  interface Props {
    src: string; alt: string; width: number; height: number;
    sizes?: string; loading?: "lazy" | "eager";
    fetchPriority?: "high" | "low" | "auto"; className?: string;
  }

  Renders <picture> with AVIF + WebP srcset at 400/800/1200/1600w
  for local public/remix/ assets. Falls through to plain <img>
  for external URLs (Wikimedia, etc.).

STEP 4 — Wire ResponsiveImage into HeroImage.tsx
  Replace internal <img> with <ResponsiveImage>.
  sizes="100vw" fetchPriority="high" loading="eager"

STEP 5 — Verify lazy boundaries on master components
  Confirm these are lazy-loaded in all 14 page components:
    SocialProofEngine, GuaranteeBlock, CapacitySignal, ObjectionSection,
    ValueLadder, MasterBuilderSeal
  Convert any static imports to lazy if found.

STEP 6 — Fix layout-trigger Framer Motion animations
  Grep for: animate={{ height, top, bottom, left, right, width }}
  Replace with GPU-safe equivalents (transform: scaleY/scaleX/translate).

STEP 7 — Add performance gates to scripts/preflight.ts

  checkImageBudgets():
    Scan public/remix/{TRADE_SLUG}/images/*.avif
    hero ≤ 180,000 bytes | card ≤ 80,000 bytes | gallery ≤ 120,000 bytes
    Print PASS/FAIL per slot. Exit 1 on any FAIL.

  checkFontPreload():
    Read index.html, confirm Space Grotesk + Jost in <link rel="preload" as="style">
    (not @import in CSS). Exit 1 if missing.

  Run both in preflight main() before existing checks.

─────────────────────────────────────────────────────────────────────
ACCEPTANCE GATES
─────────────────────────────────────────────────────────────────────
□ npm run preflight exits 0 with all performance checks passing
□ index.html has preconnect + preload for fonts (no @import in CSS)
□ index.html has preconnect for Supabase, Wikimedia, Google Maps

□ CLS score ≤ 0.1 (no layout shift on hero, editorial, before/after)
